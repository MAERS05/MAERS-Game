# 游戏介绍

## 如何操控自己

本游戏支持在浏览器控制台直接手动修改自己的状态，方便调试与测试。

### 1. 打开控制台
在浏览器中打开 `sq-du.html` 后，按 `F12` 或 `Ctrl + Shift + I` 打开控制台。

### 2. 常用调试入口
页面加载后，控制台可直接使用以下对象：

- `window.engine`：当前对局引擎实例
- `window.debugApplyState(playerId, patch)`：直接给指定角色写入状态
- `window.EffectDefs`：当前全部效果定义，方便查效果 ID

### 3. 给自己挂状态
例如给自己挂“闪避点数 -2”：

```js
window.debugApplyState('P1', { dodgeDebuff: 2 })
```

你也可以写入其它状态，例如：

```js
window.debugApplyState('P1', {
  ptsDebuff: 1,
  guardDebuff: 1,
  agilityDebuff: 1,
  insightDebuff: 1,
})
```

### 4. 给自己装效果槽
如果你要直接测试某个效果是否触发，可以给自己的行动槽挂效果：

```js
window.engine.assignEffect('P1', 'attack', 0, 'bleed')
window.engine.assignEffect('P1', 'guard', 1, 'fortify')
window.engine.assignEffect('P1', 'dodge', 2, 'depress')
```

参数说明：
- `playerId`：`'P1'` 或 `'P2'`
- `action`：`'attack' | 'guard' | 'dodge'`
- `slot`：`0 | 1 | 2`
- `effectId`：效果 ID，`null` 表示清空

### 5. 注意事项
- 控制台写入的是当前对局实例的内部状态，刷新页面会重置。
- 若你修改的是状态值，请注意它和效果槽是两回事：
  - **状态值**：如 `dodgeDebuff`、`staminaPenalty`
  - **效果槽**：如 `bleed`、`fortify`
- 若需要确认当前可用效果 ID，可以在控制台查看 `window.EffectDefs`。

---

## 游戏说明补充

- 回合中敌方就绪后的实时状态默认不完全可见。
- 若你通过洞察解锁，对手的命数、精力、动速和状态图标才会恢复实时显示。
- 效果可叠加，但是否叠加取决于该效果的实现方式与状态写入策略。
