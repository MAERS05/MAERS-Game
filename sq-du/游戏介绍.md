# 游戏介绍 (SQ-DU)

《神启·独》是一套基于"信息差、资源管理、心理博弈"为核心要素的回合制对战引擎。在游戏中，交战的胜负并非由运气（如随机数发牌或乱数伤害）决定，而是由纯粹的博弈推理和最优解构筑而成。

## 1. 核心资源
- **命数 (HP)**：决定生死的底线。当一方的命数降至 0 时战败。
- **精力 (Stamina)**：决定了战斗灵活度的行动力。所有的行动（如发动强力攻击、使用洞察窥探对手意图）都会消耗精力。精力不足时行动成本将转化为惩罚（疲劳机制）。

## 2. 行动系统 (Actions)
每回合，玩家必须在决策期内从以下行动中做出选择。攻击、守备、闪避这三种战斗行动包含「点数 (Points)」和「动速 (Speed)」两个物理维度：

- **攻击 (Attack)**：主动出击，点数大于对方会造成打击。
- **守备 (Guard)**：结下护盾进行格挡，可抵消攻击时的点伤害。
- **闪避 (Dodge)**：以极高的动速在空间轴上避开打击，动速优势下甚至可以通过点数造成反击。
- **蓄势 (Standby)**：放弃本回合的战斗动作以平息呼吸，恢复 1 点精力。蓄势不参与点数和动速的物理碰撞。
- **疗愈 (Heal)**：不消耗精力，恢复 1 点命数。疗愈同样不参与物理碰撞，是纯粹的资源恢复行为。

## 3. 时间与速度轴 (Speed / Timeline)
所有发出的动作将按"动速从大到小"排入裁判层的时间轴中执行：
- 动速快的动作（如迅闪）会率先执行完毕；
- 动速相同时发生拼刀对冲；动速不同时，快的动作可能抢先打出"差合"伤害。

## 4. 洞察机制 (Insight System)
这是本游戏最核心的情报系统，分主动与被动两种触发方式：
- **主动洞察**：玩家可主动消耗 1 格精力，强行看穿对方已选定（或将选定）的行动，以此制定克制解。
- **被动洞察（暴露期）**：回合决策拥有现实时间限制。若某方决策犹豫超过 30 秒倒计时，便会进入"暴露期"，其行动将直接对外界敞开，免费给对方提供情报！

## 5. 识破：无解的空局 (Insight Clash)
若在本回合内，**双方都看穿了彼此**（比如你主动洞察了对方，而对方发呆超时被暴露），系统将触发「识破」现象：
- *"双方心思彼此透明——任何行动在此刻都失去意义。"*
- 该回合内的所有行为强制取消，不再发生伤害互换，且本回合已提交的行动精力成本将悉数退回。
- 全场唯一不可退回的，只有玩家主动发起洞察时所提前透支的"窥探成本"。

## 6. 技能系统 (Skill System)
玩家在装备期可为每个行动最多配置 3 个技能槽位，技能在对应行动触发时产生特殊效果：

- **前置技能（onPre）**：在裁判结算前修改行动参数（如增加点数、改变动速）。
- **后置技能（onPost）**：在裁判出结果后触发（如命中后附加状态、失败后减益自身）。
- **阶段技能（onPhase）**：在特定时机触发（如回合开始、下回合行动期等）。

典型技能示例：
| 技能 | 类型 | 效果 |
|------|------|------|
| 撕裂 | 攻击/后置 | 命中后为对手附加创伤 |
| 汲取 | 攻击/后置 | 攻击成功后恢复自身精力 |
| 化劲 | 守备/后置 | 守备成功后恢复精力 |
| 崩震 | 守备/后置 | 守备成功后为对手施加沉重 |

## 7. 溢出与状态叠加 (Overflow & Alterations)
战斗环境复杂多变。状态层（包含：虚弱、力量、创伤、蒙蔽、禁锢、沉重、轻盈、坚固等）通过动态加减运算影响战斗。
尤为重要的是，游戏没有"绝对浪费"的属性：
- **伤害溢出**直接削减生命（命数负溢出）；
- **精力过载**转换为虚脱减益（精力超上限时被榨干）；
- **精力透支**转换为疲劳并造成血量反噬（尝试违规消耗精力最终导致休克）。

## 8. 胜负判定
| 结果 | 条件 |
|------|------|
| 战终 | 一方命数降至 0 |
| 处决 | 精力耗尽的一方被对手命中，遭致命一击 |
| 同归 | 双方同一回合命数均降至 0 |
| 识破 | 双方互相看穿，回合作废 |
