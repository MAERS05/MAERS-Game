# AI Agent 决策逻辑模型

独·神启（SQ-DU）拥有专设的 PVE 战斗模式（`EngineMode.PVE`），支持外置的大脑代理（AI-MAES 或其他形态的神经/启发式网络模型）直接无缝接入博弈系统。引擎专门为 AI 提供了一套"高拟真且无作弊"的观察-决策回路。

## 1. 非作弊式环境感知 (Fair Observation)
AI 在引擎中拥有与原生现实玩家完全平等的视野受限设定，杜绝直接全知内存数值：
- **公共广播监控**：AI 必须监听 Engine 分发的一切对外事件流（Action_End 结算伤害、主动洞察发起广播等）进行态势更新。
- **快照推理机制**：引擎不给 AI 提供直接访问敌方未翻牌行动的后门。它只能接收经过公开化的快照，包含双方 HP/精力比/历史行动趋势/当前效果状态等可见信息。
- **效果感知**：AI 快照同时包含自身被挂的 debuff（如攻击削减、守备削减、精力惩罚）和对手的 buff（如蓄力、守备增强），从而做出相应的行动调整，而非盲目决策。

## 2. PVE 生命周期流转
当游戏进入装备期末尾和决策期起点时，`base/engine.js` 会通过特定 Hook 调用 AI：
1. **调度发起 (`_scheduleAI()`)**：通知 AI 当前轮次环境启动。
2. **洞察阶段（8%~32% 决策期）**：AI 以极高概率（最大 90%）主动使用洞察窥探玩家意图。
3. **深度延迟仿真**：AI 并不以 `0ms` 瞬发结果，而是仿真人类读秒的威压感；兜底在决策期末段落子。
4. **即时模式加速**：当 `instant` 模式下玩家先于 AI 就绪时，引擎调用 `accelerateAI()` 取消当前长延迟，用 0.5~1.5s 快速完成决策。AI 基础决策时间不变，仅在玩家就绪后触发加速。
5. **重决策（Redecide）**：若 AI 通过洞察发现玩家已选定行动，立即触发针对性重决策，以极高概率切换至克制行动。

## 3. 核心评估算法：分层决策链

```
buildDecision()
  └── AIBaseLogic.pickAction()        ← 基础打分（快照驱动）
        ├── 精力阶段管控（4阶段）
        ├── 效果感知权重（自身/对手 buff & debuff）
        ├── 击杀窗口评估
        └── 疗愈条件评估
  └── AIJudgeLayer.buildDecision()    ← 整合决策
        ├── AIEnhanceLayer.constrainDecision()  ← 通用场景底线
        └── maesConstrainDecision()             ← MAES 专属场景锁
  └── AIExtraLayer.pickEffects()      ← 技能选择打分（B+T权重体系）
```

## 4. 精力管控系统（Phase-Based Stamina）

AI 根据自身有效精力分为四个阶段，配合精力差（`staminaGap = AI精力 - 玩家精力`）动态调整攻守节奏：

| 阶段 | 精力值 | 核心策略 |
|------|--------|----------|
| **危机** | 0~1 | 强蓄势(+4.0)；低血时零成本疗愈优先 |
| **保守** | 2 | 对手低精力→出手；对手高精力→回气；低血时加蓄势 |
| **均衡** | 3 | 精力差≥2→施压；精力劣势→偏防守 |
| **充裕** | 4+ | 全面施压，攻击+1.5；优势≥2再加攻击 |

## 5. 技能权重体系（B + T 模型）

所有技能由**基础权重（B=1.4）**和**调优权重（T）**构成，基础浮动严格控制在 ±0.2，场景加成独立叠加：

```js
// 示例（ai-extra.js）
[EffectId.DRAIN]: B + (playerLowStamina ? 0.5 : 0) + (aiLowHp ? 0.4 : 0)
// 基础 1.4，对手精力低时 +0.5，AI低血时 +0.4
```

**攻击技能**（选出权重供参考）：

| 技能 | 基础 | 主调优触发 |
|------|------|-----------|
| 撕裂 | 1.5 | 对手低血 +0.4 |
| 泣命 | 1.3 | 对手低血 +0.8 |
| 汲取 | 1.4 | 对手低精力/AI低血 +0.5/+0.4 |
| 蛮力 | 1.5 | — |

**守备技能**：

| 技能 | 基础 | 主调优触发 |
|------|------|-----------|
| 化劲 | 1.5 | — |
| 纳气 | 1.3 | AI精力≤2 +0.8 |
| 恢复 | 1.3 | AI低血 +0.6 |
| 迅防 | 1.5 | — |

## 6. MAES 专属场景底线（maesConstrainDecision）

在通用约束层之后，MAES 追加三条不可违背的场景规则：
1. **血线保命**：HP=1 且对手有精力 → 强制守备
2. **空精反守**：对手精力=0 → 禁止蓄势/疗愈，强制进攻
3. **自残保护**：HP=1 → 移除所有带 `hpCost` 的技能效果

## 7. 扩展与神经接口
所有 AI Agent 被要求遵守通用接口 `takeAction(engineCtx, snapshot)`。开发者可在 `ai-maes.js` 等外挂扩展包中，随时替入云端大语言模型推理、MiniMax 算法，或强化学习模型，以此达到训练对战 AI 的最终目的。
