# 技术架构说明 (Engine Architecture)

`sq-du` 目录下的引擎系统被严格分层，保证了复杂博弈游戏的扩展性与易测性，整体遵循"逻辑与表现分离、状态修改流水线化"的设计哲学。主要包含四大模块结构：

## 1. 裁判层 `main/judge.js` (JudgeLayer)
**定位**：纯粹的物理与规则核心（无状态的纯函数集合）。
- **职责**：不拥有任何持续性状态。它仅仅接收外部提供的一个特定「时间截面快照」，然后在内部铺设一条按动速降序的虚拟时间线（Timeline），并模拟出两人发生物理碰撞的绝对客观结果（造成了多少点数伤害、谁被谁格挡、谁成功闪避等）。
- **胜负判定**：所有胜负判定、同归于尽、处决逻辑均收敛于 `judgeGameOver`，输出唯一封包协议 `ResolveResult`。战终/处决/同归场景下仅更新 `clashName`，保留原详细战报（`clashDesc`）供 UI 展示。

## 2. 效果层 `main/effect.js` (EffectLayer)
**定位**：战术状态缓冲带与能力修饰器。
- **职责**：分为两大宏观生命周期：
  - `onPre`（裁决前的战前修饰）：把盘踞在玩家身上的各个 Effect（如锁链禁止闪避、力量加成伤害），编译重写成能丢给法官（Judge）判定的最终提交订单 `ActionCtx`。
  - `onPost`（裁决后的因果应用）：在法官判定出实质伤害后，触发特定被动（例如挨打就掉生命上限、或者受伤后叠加怒气），最终处理回合末机制（效果衰减、计时器递增）。
- **特点**：提供了极致的扩展能力。只需将新的规则配置注册进 `effect-handlers.js`，即可插拔式实现各种奇幻的战斗特质，而不需要改动引擎核心。

## 3. 总线与解析枢纽 `base/resolver.js` (Resolver)
**定位**：调度者与数据路由管道。
- **职责**：如同一位制片人，串联起 EffectLayer 和 JudgeLayer。它负责从 Engine 收到两名玩家的意图后开始处理：先丢给 Effect 进行 `onPre` 修改 → 捕捉增量（Flash Data）并塞入快照给 Judge → 接受 Judge 产出的 Timeline → 再丢给 Effect 进行 `onPost` → 将处理完毕附带上最终血量/精力计算的结果发送回 Engine。
- **特点**：处理特殊中断信号。如短路型的 `INSIGHT_CLASH`（识破废除回合）逻辑就在此判断并绕过常规战斗演算。

## 4. 引擎核心机 `base/engine.js` (Engine Controller)
**定位**：带状态的主从控制核心（State Machine Controller）。
- **职责**：维护整局游戏唯一且客观的主舞台（PlayerState）。包含回合滴答时钟（Tick Timer）、阶段状态机（配置期装备槽、三十秒死线、暴露期与结算动画期）。
- **即时模式**：引擎支持 `instant` 标记，开启后跳过回合开始/结束的 1s 延迟、装备期不启动计时器（手动关闭）、行动期缩短为 1s。通过 `startGame({ instant: true })` 启用，`skipEquip()` 提供手动关闭装备期接口。
- **特点**：提供所有供 UI 乃至外部（如网络模式）调用的动作 API。所有生命周期通过强类型的事件总线 `EventBus` 广播给外部表现层。

## 5. UI 与前端呈现 `main/ui.js` / `.html`
**定位**：响应式视图。
- **职责**：它不干涉游戏逻辑，只倾听 `EventBus` 发来的消息，利用 `DOM` API 进行面板重绘。包含浮动的状态显示框、交手日志、三段式详细 tooltip 面板等。
- **技能选择器**：内置"简"模式按钮，一键切换为三列紧凑网格布局，隐藏描述文本，适合快速选技时使用。
- **特点**：能轻松实现前端录像回放与战局恢复（通过给它灌入历史事件流和状态快照）。

## 6. AI 层 `ai/` (AI Decision Chain)
**定位**：分层的玩家行为模拟器。

```
ai-scheduler.js   ←  时机调度（决策期/洞察期/重决策期/即时加速）
  └── ai-base.js   ←  基础打分（精力阶段管控 + 双侧效果感知）
  └── ai-strategy.js ← 策略层（已知底牌/重决策克制）
  └── ai-judge.js  ←  决策构建
  └── ai-enhance.js  ← 通用场景底线约束
  └── ai-extra.js  ←  技能评分（B+T 双权重体系）
  └── sq-du-maes/ai-maes.js  ← MAES 专属技能池 + 场景约束
```

### AI 即时模式加速
即时模式下，当玩家就绪后 AI 尚未决策时，引擎调用 `accelerateAI()`（定义在 `ai-scheduler.js`）取消当前长延迟，用 0.5~1.5s 短延迟重新调度决策。AI 基础决策时间不变，仅在玩家就绪后加速。

### AI 技能权重体系（B + T）
所有技能权重 = **基础权重 B（1.2~1.6）** + **场景调优 T**，基础浮动严格控制在 ±0.2，场景加成独立叠加，保证 AI 行为的多样性与可读性。

## 7. 技能文件目录 `skill/`

```
skill/
├── attack/          # 玩家+AI 共享攻击技能（撕裂、泣命、汲取等）

├── ai-attack/       # AI 专属攻击技能
│   ├── blood_drink.js   # 饮血
│   ├── brute_force.js   # 蛮力（无条件力量，失手追加沉重）

│   ├── heavy_press.js   # 猛压（守备后力量）
│   └── pursuit.js       # 追击（闪避后轻盈）
├── ai-guard/        # AI 专属守备技能
│   └── swift_guard.js   # 迅防（守备同时附加轻盈）
└── ai-dodge/        # AI 专属闪避技能
```

## 8. 新游戏变体开发指南

当需要基于现有引擎创建新的游戏变体（如 `test.html`）时，所有文件按可复用程度分为三层。

### 🟢 直接复用（不改，import 路径指向 sq-du/ 原文件）

这些文件是纯引擎机制，不含任何游戏特定内容，**直接引用，不复制**：

| 文件 | 职责 |
|------|------|
| `base/timer.js` | 双方独立计时器 |
| `base/resolver.js` | 行为碰撞（Clash）判定 |
| `main/effect.js` | 效果管线（pts / speed / cost 计算） |
| `main/judge.js` | 结算结果构建 |
| `effect/timing.js` + `timing-constants.js` | 效果时机分发与常量 |
| `effect/status-factory.js` | 状态效果工厂函数 |
| `effect/function/*` | 基础状态效果实现（力量/虚弱/坚固等） |
| `ai/ai-base.js` | AI 评估快照与打分（马尔可夫链） |
| `ai/ai-scheduler.js` | AI 时机调度 |
| `ai/ai-judge.js` | AI 决策构建 |
| `ai/ai-strategy.js` | AI 策略层（已知底牌） |
| `ai/ai-enhance.js` | AI 场景约束（底线规则） |
| `ai/ai-manual.js` | 开发调试接管工具 |
| `effect/function/overflow-manager.js` | 溢出管理 |

### 🟡 复制后定制（每个游戏变体一份）

| 文件 | 需要修改的内容 |
|------|---------------|
| `base/constants.js` | 增删 `EffectId` 枚举值和 `EffectDefs` 技能元数据 |
| `base/effect-handlers.js` | 对应 EffectId 增删 import 和映射条目 |
| `base/engine.js` | 将 AI import 路径改为本变体的 AI 定制文件（仅 1 行） |
| `ai/<变体>/ai-<变体>.js` | 修改技能池列表和 AI 定制化属性 |
| `ai/ai-extra.js` | 若新增 AI 不认识的技能，在评分表中添加条目 |

### 🔴 必须新建

| 文件 | 说明 |
|------|------|
| `<变体>.html` | 游戏入口页面（HTML 结构、CSS、`<script>` 指向 UI） |
| `<变体>/main/ui.js` | UI 交互层（角色名、布局、效果展示逻辑） |
| `<变体>/skill/**/*.js` | 新技能实现（`onPre`/`onPost`/`onPhase` 钩子） |
| 角色立绘 / 背景图 | 视觉素材 |

### 推荐目录结构（以 test 变体为例）

```
MAERS-Game/
├── test.html                          🔴 入口（script 指向 test/main/ui.js）
├── sq-du/                             （原版，完全不动）
└── test/
    ├── base/
    │   ├── constants.js               🟡 改 EffectId / EffectDefs
    │   ├── effect-handlers.js         🟡 改技能注册表
    │   └── engine.js                  🟡 改 1 行 AI import
    ├── main/
    │   └── ui.js                      🔴 新 UI 层
    ├── skill/                         🔴 新技能（若有）
    │   ├── attack/
    │   ├── guard/
    │   └── dodge/
    └── ai/
        └── sq-du-test/
            └── ai-test.js             🟡 改技能池和定制属性
```

> `test/` 中的文件通过相对路径直接 import `sq-du/` 中的 🟢 引擎层，例如：
> ```js
> // test/base/engine.js
> import { DualTimer }     from '../../sq-du/base/timer.js';
> import { resolve }       from '../../sq-du/base/resolver.js';
> import { EffectLayer }   from '../../sq-du/main/effect.js';
> ```

### 玩家定制化

玩家技能库目前由 `createPlayerState()` 扫描 `EffectDefs` 自动填充（过滤 `aiOnly: true` 项）。如需玩家独立技能池，可参照 `ai-maes.js` 模式创建 `player-<变体>.js`，在 `engine.js` 中 `createPlayerState(PlayerId.P1)` 后调用 `applyPlayerCustomization()` 覆写 `effectInventory`。
