# sq-du 架构说明（含扩展指南）

## 1. 分层总览

### 1.1 引擎层：`base/engine.js`
职责：
- 回合状态机推进（开始/装配/决策/暴露/重筹/行动/结算/结束）
- 统一派发阶段事件
- 调用效果层与结算层
- 对外提供 API（UI/AI/联机驱动）

不负责：
- 不直接写某个效果具体规则（如“创伤如何扣血”）

---

### 1.2 效果层：`main/effect.js`
职责：
- 收集当前可触发效果
- 在阶段事件中分发执行
- 统一维护“触发判定入口”（例如攻击是否成功）

不负责：
- 不产出最终 clash 结论

---

### 1.3 效果实现层：`effect/*/*.js`
职责：
- 单效果规则实现（onPre / onPost / onPhase）
- 维护 `name` / `desc`（文案主来源）

---

### 1.4 裁判层：`main/judge.js` + `base/resolver.js`
职责：
- 计算 clash（交锋情形）
- 推演伤害与处决
- 产出 `ResolveResult`

不负责：
- 不调度效果触发策略

---

### 1.5 表现层：`sq-du/ui.js`
职责：
- 订阅引擎事件并渲染
- 转发用户输入到引擎 API

不负责：
- 不改写规则状态

---

## 2. 事件机制与时机

### 2.1 阶段事件
- `TURN_START_PHASE / TURN_END_PHASE`
- `EQUIP_START / EQUIP_END`
- `DECISION_START / DECISION_END`
- `EXPOSE_START / EXPOSE_END`
- `REDECIDE_START / REDECIDE_END`
- `ACTION_START / ACTION_END`
- `RESOLVE_START / RESOLVE_END`

### 2.2 统一效果总线
- `PHASE_EFFECT_HOOK`

### 2.3 阶段后快照同步
- `PHASE_STATE_SYNC`
- 含义：每次阶段效果执行后同步一次状态快照，避免“逻辑触发了但 UI 还没刷新”。

---

## 3. 单回合数据流（推荐理解路径）
1. 引擎触发阶段事件
2. 效果层按时机收集并执行效果
3. 引擎发 `PHASE_STATE_SYNC`（UI 刷新）
4. 裁判层产出 `ResolveResult`
5. 效果层按结果触发后置效果
6. 引擎写回状态并推进下一阶段

---

## 4. 成功/失败判定收拢

已收拢到效果层统一入口（在 `main/effect.js`）：
- 先读基础结算（`clash`）
- 再读最终伤害（效果改写后）
- 输出语义判定（`success/reason`）

用途：
- 解决“侥幸仍触发命中类效果”等问题
- 方便未来扩展免疫、护盾、转伤、一次性抵消等新规则

---

## 5. 如何扩展（不重构底层）

### 5.1 新增效果
步骤：
1. 新建 `effect/xxx/your_effect.js`
2. 实现 `id/name/desc` 与钩子
3. 在 `constants.js` 注册 `EffectId` + `EffectDefs`（结构元数据）
4. 在 `effect-handlers.js` 注册 handler

建议：
- 文案只改效果文件，减少多处同步成本

---

### 5.2 新增阶段规则
步骤：
1. 新增事件常量（如需）
2. 在引擎阶段推进处触发 `_emitPhaseEffectEvent(...)`
3. 在效果层加该阶段的分发处理
4. UI 通过 `PHASE_STATE_SYNC` 或阶段事件刷新

---

### 5.3 新增判定方式（例如“免疫一次攻击”）
不需要推翻架构，按以下做：
1. 在效果执行时写入状态标记或改写伤害
2. 扩展统一判定入口（攻击是否成功）
3. 后置效果依赖统一判定，不再自己散判

---

### 5.4 新增模式（联机/脚本）
- 复用 engine + effect + resolver
- 替换输入驱动
- UI 层无须承载规则差异

---

## 6. 职责边界检查清单（改动前后必看）
- 是否把效果规则写进了 engine？（应避免）
- 是否在 ui 中直接改游戏状态？（应避免）
- 是否出现同一效果多处触发？（要收敛）
- 是否存在“逻辑时机”和“显示时机”错位？（需 `PHASE_STATE_SYNC`）
- 文案是否集中于效果文件？

---

## 7. 现阶段约定（你当前项目）
- 效果文件是执行主体
- 效果描述文本优先来自效果文件
- 结算判定用于定义“成功/失败”
- 阶段接口均可承载效果并在阶段后刷新状态
